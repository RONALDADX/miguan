# 🚀 A股实时监控系统 - 用户使用指南

## 📋 系统概述

本系统是一个完整的A股实时监控解决方案，集成了：
- **实时数据获取**：从多个免费API获取A股实时行情
- **智能策略评估**：三个专业策略并行分析
- **风险控制系统**：自动风险评估和仓位建议
- **预警通知系统**：实时监控异常情况
- **数据持久化**：完整的历史记录和报告

### 2. 🎯 策略系统

#### 一号策略：多指标共振策略
**核心条件：**
- MACD在零线以上
- 满足以下条件之一：
  1. 绿柱逐步缩短
  2. 有金叉趋势

  3. 金叉

  4. 底背离（最佳）

**辅助条件：**
1. RSI金叉且RSI6≤68

2. BBI金叉

3. DPO金叉且角度>20°

4. OBV5天在MAOBV上方

5. KDJ金叉或有金叉趋势

6. DMI(DI1上穿ADX且ADX>20)

7. 布林线上穿中轨且三轨上行

8. 换手率≥5%

9. 成交量高于前日1倍或15日均量

#### 二号策略：DMI趋势策略
**核心条件：**
- DMI双重确认：
  1. DI1上穿DI2

  2. ADX上穿ADXR线

**辅助条件：**
- RSI金叉

- 换手率>5%

#### 三号策略：趋势启动策略
**核心条件（必须全部满足）：**
1. DMI(DI1上穿ADX)

2. RSI金叉

3. 换手率>5%

4. 布林线突破中轨

### 3. ⚠️ 风险控制系统

#### 风险评估等级
系统自动评估每只股票的风险等级：

| 风险等级 | 波动率范围 | 最大仓位 | 止损比例 |

|----------|------------|----------|-----------|

| 低风险   | <2%        | 20%      | 5%        |

| 中风险   | 2-5%       | 15%      | 8%        |

| 高风险   | 5-10%      | 10%      | 12%       |

| 极高风险 | >10%       | 5%       | 15%       |

#### 风险控制参数
编辑 `config.json` 中的 `risk_control` 部分：
```json
{
  "risk_control": {
    "max_daily_loss": 0.02,      # 单日最大亏损
    "max_position_per_stock": 0.2, # 单只股票最大仓位
    "stop_loss_ratio": 0.08,     # 止损比例
    "take_profit_ratio": 0.2,    # 止盈比例
    "min_risk_reward": 2.0       # 最小风险收益比
  }
}
```

### 4. 📢 预警系统

#### 预警类型
1. **价格突破预警**
   - 涨跌幅超过5%触发
   - 超过7%升级为警告

2. **成交量异动预警**
   - 成交量放大3倍以上

3. **策略信号预警**
   - 策略得分≥80分

4. **风险预警**
   - 波动率>8%

#### 预警配置
编辑 `config.json` 中的 `alerts` 部分：
```json
{
  "alerts": {
    "price_breakout": 5.0,      # 价格突破阈值
    "volume_spike": 3.0,        # 成交量放大倍数
    "volatility_warning": 8.0,  # 波动率警告阈值
    "strategy_signal": 80       # 策略信号阈值
  }
}
```

## 📊 数据管理

### 数据库结构

#### 1. stock_data 表（股票数据）
```sql
CREATE TABLE stock_data (
    id INTEGER PRIMARY KEY,
    symbol TEXT NOT NULL,        -- 股票代码

    name TEXT,                  -- 股票名称

    price REAL,                 -- 当前价格

    change REAL,                -- 涨跌额

    change_percent REAL,        -- 涨跌幅

    volume INTEGER,             -- 成交量

    amount REAL,                -- 成交额

    high REAL,                  -- 最高价

    low REAL,                   -- 最低价

    open REAL,                  -- 开盘价

    pre_close REAL,             -- 前收盘价

    pe REAL,                    -- 市盈率

    pb REAL,                    -- 市净率

    market_cap REAL,            -- 总市值

    circulating_cap REAL,       -- 流通市值

    timestamp DATETIME,         -- 数据时间戳

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 2. strategy_results 表（策略结果）
```sql
CREATE TABLE strategy_results (
    id INTEGER PRIMARY KEY,
    symbol TEXT NOT NULL,        -- 股票代码

    strategy_name TEXT NOT NULL, -- 策略名称

    score REAL,                 -- 策略得分

    conditions_met INTEGER,     -- 满足条件数

    total_conditions INTEGER,   -- 总条件数

    signals TEXT,               -- 信号列表

    recommendation TEXT,        -- 推荐等级

    confidence REAL,            -- 置信度

    timestamp DATETIME,         -- 评估时间

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

```python
import sqlite3
import pandas as pd
import numpy as np

conn = sqlite3.connect('stock_monitor.db')

# 获取历史数据
df_history = pd.read_sql_query('''
    SELECT symbol, price, timestamp 
    FROM stock_data 
    ORDER BY symbol, timestamp
''', conn)

conn.close()

# 计算收益率
df_pivot = df_history.pivot(index='timestamp', columns='symbol', values='price')
df_returns = df_pivot.pct_change().dropna()

# 分析策略表现
print("历史收益率统计:")
print(df_returns.describe())

# 计算夏普比率
sharpe_ratio = df_returns.mean() / df_returns.std() * np.sqrt(252)
print("\n夏普比率:")
print(sharpe_ratio)
```

## 🔧 故障排除

### 常见问题

**解决方案：**

#### 2. 数据库错误
**症状：** SQLite错误或数据库文件损坏

**解决方案：**
```bash
# 备份当前数据库
cp stock_monitor.db stock_monitor_backup.db

# 重新初始化数据库
rm stock_monitor.db
python3 -c "
import sqlite3
conn = sqlite3.connect('stock_monitor.db')
conn.execute('CREATE TABLE test (id INTEGER PRIMARY KEY)')
conn.close()
print('数据库重新初始化完成')
"

# 或者使用修复工具
sqlite3 stock_monitor.db ".dump" | sqlite3 stock_monitor_new.db
```

#### 3. 策略评估异常
**症状：** 策略得分异常或信号错误

**解决方案：**
```bash
# 检查历史数据完整性
```
